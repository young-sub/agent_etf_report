# Full Live Pre-Publish Preview Default

Accepted. Agent TReport pre-publish preview is the operator's final quality check before Telegram delivery, so its default live path validates every currently implemented ETF SourceProvider plus the current ValidatedExternalEvidenceProviderSet rather than sampling representative providers.

The current ValidatedExternalEvidenceProviderSet is Finnhub, yfinance, DART, Alpha Vantage, NewsAPI, and Naver. SEC EDGAR is a KnownUnvalidatedProviderException for this closure because it has rate-limit/block evidence but no successful live smoke evidence. SEC EDGAR remains disclosed in preflight, approval, handoff, and documentation evidence, but it is not requested in this goal and does not count as a full user-ready closure blocker until a separate successful live smoke promotes it into the validated required set.

The risk of broader external calls is constrained by the daily approval boundary, vendor-policy-aware pacing, a default 25 target maximum, provider-specific lower caps, scratch-only smoke approval profiles, provider failure exclusion evidence, a 600 second default preview timeout with explicit overrun override, and preserving Telegram delivery as not sent.

The final handoff separates artifact closure from full user-ready closure. Artifact closure requires generated report artifacts plus the TelegramSignalAlert artifact and message body in `pre_publish_handoff.json`. Full user-ready closure additionally requires readiness, required provider/API outcomes, and quality evidence to allow delivery. Provider `no_data` is user-ready-compatible when it comes from a normal provider response; credential-required, blocked, rate-limit exhaustion, provider-unavailable, invalid-payload, or timeout-exhausted outcomes block full user-ready closure only for required providers. Provider/API failures can still produce `operator_review_only` artifacts when safe, while readiness failure, report generation failure, timeout, or error-severity quality failure must not expose the Telegram body as send-ready. Final pre-publish handoffs keep operator artifact references path-safe: relative artifact paths are allowed for local inspection, but file URIs, absolute local paths, raw provider URLs/endpoints, credentials, and Telegram Bot API delivery targets are not.

Provider-specific lower target caps are allowed inside the global 25-target maximum when official provider policy requires them. Capped targets are coverage limitations, not hidden omissions. Alpha Vantage `NEWS_SENTIMENT` uses one grouped ticker request for the capped target subset and records the cap in `provider_limitations`; it must not degrade into one request per target in the pre-publish default path. Successful or normal no-data external evidence may be reused only inside the same approved smoke boundary with matching run id, approval boundary fingerprint, holdings/report target identity, requested provider set, and evidence category. This is not a cross-run evidence database.

Final pre-publish preview writes a path-safe result package under `data/agent_treport/live-source/daily-smoke-summaries/<run_id>/` by default. The package contains safe handoff and summary copies, approval/preflight summary, external evidence summary, provider exception summary, validation-command result placeholders, canonical-history non-mutation evidence, and retention evidence. It must not include raw provider payloads, raw URLs/endpoints, credentials, environment values, absolute paths, file URIs, stack traces, or Telegram Bot API delivery targets.

SEC EDGAR can become a required provider only in a later goal after a separate live smoke returns success or normal no-data for at least one valid target using SEC-policy-compliant pacing and leaves path-safe evidence with no active cooldown or blocked outcome. Promoting SEC EDGAR, or otherwise expanding the required provider set, is provider expansion and requires approval boundary renewal.
